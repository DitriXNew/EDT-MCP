# Правовой статус и интероперабельность

> Это не юридическая консультация. Для значимых (особенно коммерческих) решений
> необходим анализ конкретного лицензионного соглашения 1С профильным юристом.

## Цель работы

Материалы в папке `docs_api-edt` создаются с **единственной целью — обеспечение
взаимодействия (интероперабельности)** независимо разработанного плагина
`com.ditrix.edt.mcp.server` (лицензия AGPL-3.0-or-later) с платформой 1C:EDT.

## Что делалось и что НЕ делалось

Делалось (инспекция ради совместимости):
- чтение **публичных сигнатур** классов через `javap -p` (имена методов, параметры);
- чтение `META-INF/MANIFEST.MF` (экспортируемые пакеты и их версии);
- анализ байткода для понимания **последовательности вызовов** API;
- фиксация **сигнатур и фактов поведения** своими словами.

НЕ делалось:
- копирование исходного кода 1С (реализаций) в репозиторий;
- распространение `.jar`-файлов 1С или декомпилированных исходников;
- создание конкурирующего аналога продукта.

## От чего зависит правомерность

1. **EULA на 1C:EDT** — наличие/формулировка запрета на декомпиляцию и реверс-инжиниринг
   (определяющий документ, требует отдельной проверки).
2. **Юрисдикция** — применимые исключения для интероперабельности.
3. **Характер действий** — чтение сигнатур vs полная декомпиляция логики vs копирование кода.

## Зоны пониженного риска

- **API как факты.** Имена и сигнатуры методов, как правило, относятся к неохраняемой
  идее/факту, а не к охраняемому выражению. Зафиксированы именно сигнатуры, не реализация.
- **Нет распространения кода 1С.** Только описания и собственные иллюстративные примеры.
- **Разработка плагина против API EDT — штатный сценарий** (EDT построен на Eclipse;
  расширение через `Import-Package` — обычная практика).

## Зоны повышенного риска

- **Договорный запрет в EULA.** Если соглашение 1С прямо запрещает декомпиляцию/реверс —
  это договорное ограничение; его соотношение с законными исключениями зависит от
  юрисдикции. Требует проверки текста EULA.
- **Внутренние пакеты `com._1c.g5.v8.dt.internal.*`** — не публичный API: не
  поддерживаются, могут меняться без предупреждения, выше вероятность ограничений на
  использование. В планах присутствует зависимость от `com._1c.g5.v8.dt.internal.form`
  (доступ к Guice-инжектору) — отмечать как риск.
- **Глубина анализа байткода** — чтение сигнатур мягче, чем полная декомпиляция логики.

## Исключения для интероперабельности (справочно, не исчерпывающе)

- **РФ — ст. 1280 ГК РФ:** изучение/декомпиляция для достижения **взаимодействия**
  независимой программы при условиях: использовать только необходимую информацию, не для
  создания аналога, не передавать третьим лицам. EDT — продукт российской компании,
  поэтому норма, вероятно, релевантна.
- **ЕС — Директива 2009/24/EC, ст. 5–6:** аналогичное исключение при условиях.

## Правила снижения риска (применять при ведении этой папки и кода)

1. Сверяться с актуальным EULA 1C:EDT (раздел про reverse engineering / decompilation).
2. Предпочитать **экспортируемый публичный API** (например `com._1c.g5.v8.dt.form.generator`)
   зависимостям от `internal`-пакетов; опору на `internal` помечать как риск.
3. Не хранить и не распространять артефакты 1С и декомпилированные исходники — только
   сигнатуры и собственные заметки.
4. Хранить в документации только то, что необходимо для совместимости.
5. При публикации/коммерческом использовании — консультация юриста по конкретному EULA.
