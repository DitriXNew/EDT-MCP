# docs_api-edt (служебная папка)

> Назначение: фиксация разобранного API платформы 1С:Enterprise Development Tools (EDT),
> которое используется (или планируется к использованию) в MCP-сервере
> `com.ditrix.edt.mcp.server`. Это **служебная** документация для разработки,
> а не пользовательская справка по продукту.

## Зачем это нужно

EDT не публикует официальную справку по большинству внутренних сервисов. Поэтому
здесь фиксируются результаты реверс-инжиниринга (сигнатуры классов через `javap`,
анализ байткода мастеров EDT, экспорт пакетов из `MANIFEST.MF`). Цель — чтобы
повторно не разбирать одно и то же при добавлении новых инструментов.

## Правила ведения

- Документируем только **проверенный** API (сверенный с реальными `.jar` через `javap`/анализ байткода). Непроверенные предположения помечаем явно: `(не проверено)`.
- Для каждого API фиксируем источник: имя плагина и версию (например `com._1c.g5.v8.dt.form_31.1.4`), чтобы было видно, к какой версии EDT относится сигнатура.
- Сигнатуры приводим в нормализованном виде (как выводит `javap -p`), без байткода.
- Кодовые примеры — иллюстративные (псевдо-Java по образцу существующих инструментов), а не дословный декомпилированный код EDT.

## Версия EDT, к которой относятся записи

- EDT: `2025.1.5` (дистрибутив `1c_edt_distr_offline_2025.1.5`)
- Плагин форм: `com._1c.g5.v8.dt.form_31.1.x`, `com._1c.g5.v8.dt.form.ui_24.x`
- Экспортируемый пакет генератора форм: `com._1c.g5.v8.dt.form.generator;version="4.0.0"`

Источник `.jar`: `~/.p2/pool/plugins/`.
